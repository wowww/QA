| summary | 이슈 | 수정 예시 |
| -------- | -------- | -------- |
| aria-hidden | 비선택 슬라이드 indicator 버튼에 aria-hidden="true"가 적용되어 스크린 리더로 버튼의 정확한 내용을 읽거나 기능을 사용할 수 없음.(모바일 디바이스에서는 focusing 이 되지 않음.)     | indicator 버튼은 interaction 요소이므로 aria-hidden="false"로 처리하거나 aria-hidden 속성을 삭제해야 함.  |
|aria-selected |indicator 선택 정보를 aria-selected로 제공하였으나 부적절함.1. aria-selected 속성이 동적으로 변경 안됨. 2번째 indicator가 선택 되었는데aria-selected=“true”는 첫번째 indicator에 적용되어 있음.2. aria-selected가 role=“presentation”인 `<li>` 태그에 제공 되어 스크린리더가 selected 정보를 정상적으로 읽어줄 수 없음.|1. 버튼 선택 상태에 따라 aria-selected=“true/false”가 동적으로 변경되어야 함.2. aria-selected는 선택 요소를 나타내는 적절한 role과 함께 사용해야 하며, focusable 요소에 사용해야 스크린리더가 정상적으로 읽어 줌.`<li>` 내의 `<button>`을 role=“tab”으로 변경하고 aria-selected의 위치를 `<button>`으로 수정해야 함.|
|role="listbox", role="option"|슬라이드를 role="listbox", role="option"으로 설정하여 스크린리더가 슬라이드 컨텐츠를 하나의 텍스트그룹으로 인식 함. 스크린리더가 버튼 요소를 인식하지 않아 선택 및 사용할 수 없음.대상 : ios, chrome|role="listbox", "option" 외 다른 코드 적용.|
||적절하지 않은 role이 제공됨.|목적에 맞는 role 을 제공해야 함.Ex) gnb  : role=“menubar”gnb 아이템들  : role = “menuitem”sub_gnb를 가질 경우 속성 : aria-haspopup =“true”sub_gnb: role=“menu”참고)  https://www.w3.org/TR/wai-aria-1.1/#menubar |
|aria-live="polite"|슬라이드 전체 영역에 aria-live=“polite” 속성이 적용되어 슬라이드를 이동할 때, 가장 첫 번째 컨텐츠부터 마지막 컨텐츠까지 모든 내용을 읽어 줌.|방안 1) aria-live=“off” 로 제공 또는 aria-live 속성을 삭제하거나<br>방안 2) 슬라이드가 이동되었을 때 화면에 보이는 (현재 선택 된) 영역만 스크린 리더가 읽어줄 수 있도록 수정해야 함.|
|role="tablist", role="tab" 함께 쓰이지 않은 경우|role=“tab” 속성이 role=“tablist”와 함께 사용되지 않음.|role=“tab” 속성과 role=“tablist” 속성을 함께 제공해야 스크린 리더가 aria-selected=“true” 속성과 슬라이드 개수 정보를 정상적으로 읽어 줌. 표시된 `<ul>`에 role=“tablist” 적용.|
